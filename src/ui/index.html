Bounded Signal Review and Trade-Evaluation Workflow (Non-live)

This page has one bounded workflow: review technical signals and evaluate trade candidates through non-live runtime evidence. The workflow remains technical-only and does not assert trader validation, operational readiness, or production readiness.

  1. Run bounded analysis.
  2. Define bounded watchlist scope.
  3. Evaluate ranked signal output.
  4. Inspect backtest artifacts and runtime evidence.

Technical signal visibility is explicitly separate from trader validation and operational readiness decisions.

Consolidated Bounded Operator Workflow

This single bounded operator workflow consolidates the existing watchlist, analysis, decision review, and backtest evidence surfaces into one ordered non-live flow on canonical /ui. Each step links into an existing section that consumes a canonical read or runtime surface; no new execution logic, broker integration, strategy optimization, or live-trading capability is introduced.

  1. Step 1 · Watchlist: Configure the bounded watchlist scope through canonical /watchlists read/write surfaces (no client-only watchlist state).
  2. Step 2 · Analysis: Run bounded non-live analysis through POST /analysis/run (single symbol) or POST /watchlists/{watchlist_id}/execute (ranked watchlist) consuming the existing canonical analysis routes.
  3. Step 3 · Decision Review: Review reviewed signals on the bounded technical decision surface fetched from GET /signals/decision-surface with explicit decision_state, qualification_state, missing criteria, and blocking conditions.
  4. Step 4 · Backtest Evidence: Inspect governed bounded backtest artifacts through GET /backtest/artifacts and GET /backtest/artifacts/{run_id}/{artifact_name} for non-live evidence context.

Consolidated workflow boundary: this consolidation does not introduce live trading, broker execution controls, broker integration, strategy optimization, trader validation, or operational-readiness inference. Technical signal and backtest visibility remain explicitly separate from trader validation and operational readiness decisions.

Canonical read surfaces consumed by the consolidated workflow: /watchlists, /watchlists/{watchlist_id}, /watchlists/{watchlist_id}/execute, /analysis/run, /signals/decision-surface, /backtest/artifacts, /backtest/artifacts/{run_id}/{artifact_name}.
